[CUET Inter University Programming Contest 2025 D The AND, The OR, and The XOR](https://codeforces.com/gym/106259/problem/D)

 ~~注意到~~最多只能选两个喵

 因为喵，&保留的是所有数字都有的 1，但是|保留的是只要出现过的 1，所以说，&是|的子集喵。所以说，令&完之后数字是 $a$ ,在|完后数字为 $b$，现在要做的就是让 $b - a$ 最小。

如果多一个人喵 $(a_1 \And a_2 \And a_3) \leq (a_1 \And a_2)$ 而且还有 $(a_1 | a_2 | a_3) \geq (a_1 | a_2)$ 多了一个数的话，就会让 $b$ 更大 $a$ 更小喵，这么看来，多加上一个数字怎么想都不会让当下的情况更好喵

然后喵，我这里是使用 $a \oplus b = (a \And \overline{b}) | (\overline{a} \And b)$ 来化简的喵，到最后就有 $(a_1 \And a_2) \oplus (a_1 | a_2) = a_1 \otimes a_2$ 这个结论了喵

不过这其实完全可以从“集合”的角度更优雅地解释喵：

既然 $(a_1 \And a_2)$ 是 $(a_1 | a_2)$ 的子集，那 OR 里面的 1 必然完全包含了 AND 里面的 1 喵。

它们俩做异或，本质上就是把 OR 里“多出来的那部分 1”给单独挑出来。这多出来的是什么？不就是两个数字本来就不一样的位嘛！所以结果理所当然就是 $a_1 \oplus a_2$ 喵！

最后，这道题就变成了一个在里面找两个数字异或和最小了喵

~~注意到相邻两项的异或和一定最小喵，所以，最后也只要排一下序，遍历一遍数组就好了喵~~

其实是因为因为从小到大排序后，数字在二进制的高位上相同的（也就是前缀最相似的）必然会紧紧挨在一起喵。

异或想要得到最小值，首要任务就是把高位的 1 给消掉（变成 0）。所以，只有找排序后高位最相似的相邻元素，才最有希望得到最小的异或值喵！
```cpp
void solve()
{	
	int n;
	cin>>n;
	vector<int> a(n);
	for (auto& i : a) cin>>i;
	ranges::sort(a);
	int minn = INF;
	for (int i = 1 ; i < n ; i++)
	{
		minn = min(minn,a[i] ^ a[i - 1]);
	}
	cout<<minn<<'\n';
	return;
}
```
