不知道为什么，这种背包变一下就不会写了喵，dp的状态设计也是一个难点喵

这题有两个难点喵，一个是贪心，一个是dp喵

你直接去写状态转移肯定会乱掉，因为你不知道后面挂饰的挂钩到底够不够用，对吧喵？

你想想嘛，如果先把没有挂钩的挂饰挂上去，手机上唯一的挂钩就被占用了，后面那些有很多挂钩的挂饰不就没地方挂了吗？连这点都想不到，真是笨死了喵！

所以我们需要结合一点贪心的思维，先按照挂饰能提供的挂钩数量 $A_i$ 从大到小排序喵！这样就能保证，只要你的挂钩总数够用，就绝对不会出现中途没地方挂的尴尬情况喵。处理完有挂钩的，再去处理没有挂钩的，思路不就顺了嘛喵！

状态转移其实超级简单喵！把挂饰排好序之后，这题就变成了一个纯粹的背包问题喵。设 $dp[i][j]$ 为考虑了前 $i$ 个挂饰，当前还剩下 $j$ 个空挂钩时的最大喜悦值喵。（因为你最多只有 $N$ 个挂饰，所以挂钩数量上限只要算到 $N$ 就行了，超过 $N$ 的部分对我们没有任何意义，多出来的全当做 $N$ 处理，懂了吗笨蛋喵？）

- 初始状态： $dp[0][1] = 0$ ，也就是一开始手机自带 $1$ 个挂钩，什么都没挂的时候喜悦值是 0。其他的状态全设为极小值 $-\infty$ 喵。
- 转移方程：对于第 $i$ 个挂饰，只要当前有挂钩（即 $j \ge 1$ 且状态合法），你就可以选择挂或者不挂喵：
  - 不挂：直接继承上一层的状态，什么都不做喵。
  - 挂：挂上去之后，消耗 $1$ 个挂钩，但新增了 $A_i$ 个挂钩，所以挂钩数量变成了 $\min(N, j - 1 + A_i)$。那么转移就是用当前阶段的 $dp[j] + B_i$ 去更新下个阶段的新状态喵！

```cpp
void solve()
{	
    int n;
    cin>>n;
    struct data
    {
        int nxt = 0,val = 0;
    };
    
    vector<data> a(n);
    for(auto& [ct,v] : a)
    {
        cin>>ct>>v;
    }
    ranges::sort(a,greater{},&data::nxt);
    vector<vector<int>> dp(n + 1,vector<int>(n + 1,-INF));
    dp[0][1] = 0;
    for (int i = 0 ; i < n ; i++)
    {
        auto [ct,v] = a[i];
        for (int j = 1 ; j <= n ; j++)
        {
            if (dp[i][j] != -INF)
            dp[i + 1][min(n,j + ct - 1)] = max(dp[i][j] + v,dp[i + 1][min(n,j + ct - 1)]);
            dp[i + 1][j] = max(dp[i + 1][j],dp[i][j]);
        }
    }
    int ans = -INF;
    for (const auto& i : dp)
    {
        for (const auto& j : i)
        {
            ans = max(ans,j);
        }
    }
    cout<<ans<<'\n';
    return;
}
```
