**排列 DP (Permutation DP)** 是一类专门解决“**求满足特定条件的 $1$ 到 $N$ 的排列个数**”或者“**在排列上进行最优化**”的动态规划问题。

它的核心难点在于：排列里的数字是 $1$ 到 $N$ 不重复的。如果你只是简单地枚举每一位填什么，很难处理“数字不能重复使用”这个限制，状态压缩 ($2^N$) 又只能做到 $N=20$ 左右。

而 **排列 DP** 通常能做到 $O(N^2)$ 甚至 $O(N^3)$，解决 $N$ 在 $100$ 到 $5000$ 左右的问题。

它主要有两种经典的思考流派，给我竖起耳朵听好了，喵！

### ---

**第一流派：按位置填数字 (从小到大位置) —— 关注“相对排名”**

这就是刚才那道题的思路。我们按照排列的下标 $1, 2, \dots, N$ 的顺序，一个一个位置去填数字。

* **核心痛点**：当我们填第 $i$ 个位置时，我们不知道哪些数字已经被用过了，哪些没被用过。  
* **解决方案**：我们**不关心具体的数值**，只关心当前填进去的数，在**已经填进去的这 $i$ 个数**里排老几（相对排名）。

**状态定义通常是：**

$$dp[i][j]$$  
表示：已经填好了前 $i$ 个位置，且第 $i$ 个位置上的数字，在这是 $i$ 个数中**从小到大排第 $j$**。

**转移逻辑：**

当我们要填第 $i+1$ 个数时，假设它的相对排名是 $k$。

那么在原来的 $i$ 个数中：

1. 原排名 $< k$ 的数，数值不变。  
2. 原排名 $\ge k$ 的数，数值实际上被“挤”大了一位（为了给新来的 $k$ 腾位置）。

**适用场景：**

* 题目限制的是相邻位置的大小关系（如波浪形数组、上升下降序列）。  
* 刚才那道题（给 < \> 符号限制）就是最典型的例子。

### ---

**第二流派：按数值填位置 (从小到大数值) —— 关注“连续段”**

这个稍微难一点，笨蛋主人脑子转不过来也不怪你。

这个流派是：我们按照数值 $1, 2, \dots, N$ 的顺序，把它们插到排列的空位里去。

* **核心痛点**：当我们放入数字 $i$ 时，我们不知道它最终在排列的哪个下标，只知道它比之前放进去的 $1 \dots i-1$ 都要大。  
* **解决方案**：关注数字构成的**连续段 (Connected Components)**。  
  因为我们是从小到大放数字，所有已经放进去的数字（比如 $1, 3, 4$）在最终的排列里可能是不相邻的，它们形成了若干个“连通块”或“段”。新的数字 $i$ 可以放在某个段的旁边（合并段），或者单独成为一个新的段。

**状态定义通常是：**

$$dp[i][j]$$  
表示：已经把数值 $1 \dots i$ 放进去了，它们在当前的排列中形成了 **$j$ 个不相邻的连续段**。

**转移逻辑（以插入 $i+1$ 为例）：**

数值 $i+1$ 比当前所有的数都大，所以：

1. **合并两个段**：把 $i+1$ 填在两个段中间的空位，段数 $j \to j-1$。  
2. **贴在一段旁边**：把 $i+1$ 填在某个段的左边或右边，段数 $j$ 不变。  
3. **自成一段**：把 $i+1$ 填在不与其他段相邻的空位，段数 $j \to j+1$。

**适用场景：**

* 题目限制和“绝对差值”有关，比如 $\sum |p_i - p_{i-1}|$。  
* 或者限制波峰、波谷的数量。

### ---

**举个栗子：欧拉数 (Eulerian Numbers)**

这是一个经典问题：求 $1 \dots N$ 的排列中，恰好有 $k$ 个**上升**（即 $p_i < p_{i+1}$）的排列有多少个？

这就用的是 **第一流派（从小到大位置插入）** 的变种：

我们不需要记录最后一个数排第几，只需要知道目前构成了几个上升。

当我们把数值 $i$ 插入到 $1 \dots i-1$ 的排列中（无论插在哪里，它都是最大的）：

1. 插在原来的上升位置或者最末尾 \-\> 上升数不变。  
2. 插在原来的下降位置或者最前面 \-\> 上升数 $+1$。

这比数位 DP 还要抽象一点，因为它利用了排列的**相对性**。

### **总结一下喵**

| 特性 | 数位 DP (Digit DP) | 排列 DP (Permutation DP) |
| :---- | :---- | :---- |
| **处理对象** | 一个很大的数字 (e.g., $10^{18}$) | 一个序列的排列 (e.g., $1 \dots 5000$) |
| **核心限制** | 不能超过上界，前导零 | 数字不能重复，相对大小关系 |
| **状态设计** | $pos, limit, lead, sum \dots$ | $i$ (个数), $j$ (排名/段数) |
| **复杂度** | $O(\log N \times \text{state})$ | $O(N^2)$ 或 $O(N^3)$ |
