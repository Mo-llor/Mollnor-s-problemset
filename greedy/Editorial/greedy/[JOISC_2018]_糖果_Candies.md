我原本以为反悔贪心就只是简单的吧之前贪过的数据放进一个优先队列里面，如果后面找到更优的就会把前面最劣的答案丢掉，但是很显然，这道题不仅仅是这样子的喵

反悔贪心的优先队列是要有的喵，这题也不例外，但是这道题里面存的是什么喵？直接存之前贪过的对应值吗？这显然不对。因为题目中有说，我们不能拿相邻的糖果，
如果我们之后反悔的话，还要考虑怎么把它左右的糖果拿上，这样的实现写起来会很有难度的喵

所以，我们就要另辟蹊径喵

我们要找到一种**构造优先队列数据**的方法，让我们**只要反悔贪心**就好了

那这样的话，我们考虑到，丢掉当前的元素，同时选上两边的元素一定是最优的，那么我们就能想到当优先队列里面存储 **$a_{i - 1} + a_{i + 1} - a_i$** 的时候，能够做到选取如果丢掉之前的糖果，就能够选取它旁边两个糖果喵

# 考虑选第二个糖果

我们选一个糖果肯定选的是是全局最大的那颗喵，那如果我们要再选一个喵，就只有两种情况喵

1. 选择一个次大的，和第一个糖果不相邻的糖果
2. 选和第一个糖果相邻的那两个

作为这两个选择的评判标准就是要去找就是看看这两个**哪个更大**那不就是看优先队列里面哪个先弹出来就取选哪个吗？

那这样的话，优先队列里面存的就应该是 **$a_{i - 1} + a_{i + 1} - a_i$** 喵

# 考虑贪心的正确性

我当时就在想，如果要这样做的话，为什么不会出现我已经选过 $a_{i - 2},a_{i + 2}$ 了，再出现丢掉 $a_i$ 去选 $a_{i - 1},a_{i + 1}$ 的情况喵？想到我们维护的信息喵，我们一定是先有 $a_{i - 1} + a_{i + 1} - a_i$ 再有考虑 $a_{i - 2},a_{i + 2}$ 的情况喵，所以不会有后发先至的情况喵

再观察

$$a_{i - 1} + a_{i + 1} - a_i$$

如果我们发现不选 $a_{i - 1},a_{i + 1}$ 的情况更优的话，我们就一定会选择 $a_{i - 2},a_{i + 2}$ 那么我们，再结合我们要在优先队列里面维护的数据，发现我们有：

$$a_{i - 2} + a_{i + 2} - (a_{i - 1} + a_{i + 1} - a_i) = a_{i - 2} + a_{i + 2} + a_i - (a_{i - 1} + a_{i + 1})$$

之前已经加上去的贡献居然被减掉了!!!这个式子的本质，就相当于我们在逻辑上吐出了 $a_{i-1}$ 和 $a_{i+1}$，同时吞下了更外层的 $a_{i-2}$ 和 $a_{i+2}$，并且重新找回了一开始吐掉的 $a_i$！这就完美地实现了多重套娃式的反悔，而且每次操作吃糖果的净增量永远是 1 喵！

利用好这个性质，我们就可以维护一个全局的 **sum** 喵。因为相邻的点都是异号的喵，所以**绝对不会出现选相邻的点的情况**喵。

# 关于实现喵

如果只是单纯地在数组上模拟的话，实现会很难很难喵，我们要考虑的东西会有很多很多喵。那有什么办法能够减少我们写代码的难度和思维量喵？

那当然就是用链表写了喵，因为考虑到优先队列里面存的数据，我们在链表里面维护的数据应该也要和优先队列里面的数据一致喵。我们可以这样想，在 i 的位置创建一颗魔法糖果，它的美味值是 $a_{i - 1} + a_{i + 1} - a_i$ 喵，然后删掉 $i - 1, i + 1$ 喵，这样的话，就与我们的优先队列完美适配了喵

还有喵，不要用list喵，如果我们传的是迭代器的话，我们一删除元素，优先队列里面所有的迭代器都会失效喵，不仅会导致极大的常数开销，更致命的是，底层节点被删除后会导致映射的迭代器失效喵，引发内存错误，很坏喵。

所以，我们可以用数组模拟链表喵，这样我们就可以维护一个del数组来记录当前节点有没有被删除，如果之后优先队列顶上的那项已经被标记删除了，那就直接pop就好了喵

另外就是，为了防止边界上的点在计算魔法公式时产生“无中生有”的反悔，必须在数组两端设立两个极小值的虚拟边界 $A_0 = -\infty$ 和 $A_{N+1} = -\infty$ 喵！这样的话，我们就永远不会选到不存在的断点了喵

void solve()
{
    int n;
    cin>>n;
    struct data
    {
        int l = 0,r = 0,val = 0;
    };
    vector<data> a(n + 2);
    for (int i = 1 ; i <= n ; i++)
    {
        cin>>a[i].val;
        a[i].l = i - 1;
        a[i].r = i + 1;
    }
    a[0].l = 0;
    a[0].val = -INF;
    a[0].r = 1;
    a[n + 1].l = n;
    a[n + 1].val = -INF;
    a[n + 1].r = n + 1;
    priority_queue<pll> prpr;
    for (int i = 1 ; i <= n ; i++)
    {
        prpr.emplace(a[i].val,i);
    }
    bitset<2000000> del;
    int sumn = 0;
    for (int i = 1 ; i <= (n + 1) / 2 ; i++)
    {
        while(!prpr.empty() && del[prpr.top().second])prpr.pop();
        auto [val,idx] = prpr.top();
        prpr.pop();
        sumn += val;
        auto [le,re,v] = a[idx];
        del[le] = 1;
        del[re] = 1;
        v = a[re].val + a[le].val - v;
        a[a[le].l].r = a[le].r;
        a[a[re].r].l = a[re].l;
        a[idx] = {a[le].l,a[re].r,v};
        prpr.emplace(v,idx);
        cout<<sumn<<'\n';
    }
    return;
}
